# SPACE GLEAM Design MD

## Source

- Site: https://spacegleam.co.jp/
- Core message: アイデアを、最短距離で事業にする。
- Proposal message: 小さく作って、早く検証する。

## Tone

SPACE GLEAMの資料は「開発会社の機能説明」ではなく、「事業化までの距離を短くする提案」に見せる。

- 余白を広く取る
- 黒、白、生成りを基調にする
- 強調色はブロンズ系に限定する
- 文字量を詰め込まず、1ページ1メッセージにする
- 機能名より、課題、検証、リリース、改善を前面に出す
- 数字は大きく見せる

## Palette

- Black: `#050509`
- Deep surface: `#0b0b12`
- Text: `#111111`
- White: `#ffffff`
- Warm background: `#f9f7f4`
- Light section: `#f6f6f6`
- Muted text: `#555555`
- Soft text: `#888888`
- Line: `rgba(0,0,0,0.10)`
- Bronze accent: `#ae8e71`

## Typography

- Sans: Inter, Noto Sans JP
- Serif accent: Playfair Display, Noto Serif JP, Yu Mincho
- Large title: short Japanese headline, 28〜38pt
- Section title: 20〜26pt
- Body: 9〜12pt
- Eyebrow: uppercase, letter spaced, muted

## Layout

横型プレゼンPDFはLPセクションのように構成する。

1. Hero cover
   - 黒背景
   - 左に大きな日本語タイトル
   - 右に診断カード風の白カード
   - CTA風の黒/白ボタン表現

2. Problem
   - 生成りまたは黒背景
   - 「こんな状態で止まっていませんか？」の文脈
   - 3つの課題カード

3. Proposal
   - 白背景
   - 「小さく作って、早く検証する。」
   - 初期スコープをカード化

4. Roadmap
   - 5ステップ: 仮説整理 / MVP設計 / 開発 / 検証 / 改善・拡張
   - サイトのSERVICESセクションに寄せる

5. Pricing / Next Action
   - 生成り背景
   - 概算を大きく表示
   - 「まずは小さく作って、反応を見る。」

## Do

- 「AI」「MVP」「業務システム」を事業視点で説明する
- 25万円〜、2〜4週間、小さく検証などのサイト文脈を使う
- 提案の最後は無料相談、要件整理、初期スコープ確定に接続する

## Don't

- ただの仕様書にしない
- CSV表のように全パターンを並べない
- 原色を多用しない
- 各パターンで色を変えすぎない
- 細かい要件を1枚に詰め込みすぎない
